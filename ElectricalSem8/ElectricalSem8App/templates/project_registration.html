Database Project: (deadline: 30/5/2021)

{% csrf_token %}

Project Aim and Description:

The aim of this project is to specify, design, implement, populate and query a limited-scope relational database. This constitutes a very important practical exercise to better understand all what you have taken during the “introduction to database” course. The objective of this project is to understand the main phases of the development of a relational database, and to produce the deliverable(s) of each phase: a requirement analysis document, Entity-relationship Diagram, Relational Schema creation SQL script, Data population SQL script and Main queries SQL script. The project can be done individually or in teams of two students maximum. The size and scope of the project depends on the number of students.


Project steps:

1) You need first to form your team (if needed) and then to select one of the projects in the project list at the end of this file. You should make three choices in order to solve any conflict in students choices. You are also free to choose a project from outside the list.
2) Once assigned a project, the first and very important step is to do the needed re/search about the topic/application you are intended to develop. This will help you to define the main requirements for your project; in other words to play the customer role and present the information storage problem which includes mainly what “the customer” is planning to store in the database, the business rules/ constraints to respect as well as the main reports (queries) he is expecting to have.
3) The next step is to draw the Entity-Relationship diagram for the database which will be then translated into a relational schema to be implemented in SQL in a database creation script.
4) The database should be then populated with the minimum necessary data (in a SQL script) that will help to validate the SQL queries you will write and which will help providing the reports required by the customer in the first phase.


Project Scope and Complexity:

The scope and thus the size as well as the complexity of the project is very important and will play an important role in the assessment and thus the grading of the project especially for team projects. On the other hand, it is very important to have a “reasonable” complexity level in order to be capable to finish the project on time (before the end of May 2021). Thus a main challenge is to define a reasonable scope for the project; thus in some project topics, student should only consider certain modules to be analyzed and then implemented. Due to the importance of the aspect of the project, you should get the approval of your tutor regarding the scope before proceeding; thus you should prepare a one to two pages concerning the requirement specifications that clearly demonstrates that your project is “acceptable" and “tangible" at the same time


Project Deliverable(s):

• At the end of the project, you are expected to provide a brief report that explains the different phases of the project along with the three SQL script parts (can be merged in one SQL file) sufficiently commented.
• You are also expected to do an oral presentation (Slides and demo) in order to demonstrate your understanding of the project (the date is to be determined later).

Optoelectronics Project: (deadline: 25/5/2021)



Project details

The project consists of studying and simulating the modale propagation in a multimode fiber optic with comparison between steo index and graded index fibers. It involves the following tasks:
  1. Simulate the number of guided modes versus normalized frequency V or wavelength
  2. Simulate the power in core of the first two modes versus V and lambda
  3. Simulate the power coupled into guided nides versus V and lambda
  4. The three simulations shall be done for two types oof fiber, step index and graded index
  5. Provide simulation results with comparison and discussions


Given Values:

To perform simulation, you will be given the following parameters:
  1. Core refractve index
  2. Cladding refractive index
  3. Fiber core radius
  4. The waist radius of incident Gaussian laser beam


Output:

The simulation results shall include the following:
  1. Table and graph of the variation of M vs V or lambda
  2. Table and graph of the variation of the Power in core of the first two modes vs V or lambda
  3. Table and graph of the variation of the power coupled into guided modes vs V or lambda

    4. These results shall be provided for the two types of fiber.
    RP fiber calculator will be used for simulation.


Report:

The project report shall include the following part:
  1. Small briefing on theory of guided modes in Fiber Optic
  2. Simulation screen shots
  3. Simulation results with tables and graphs
  4. Comparison and discussion
  5. Conclusion

    6. The total number of pages must not exceed 20 pages.


Notes:

  1. The reports shall be sent by email
  2. PPT presentation shall be prepared to present the report
  3. Each group will have 10 minutes to present the project and 10 minutes for questions and answers (20 minutes in total).
  4. A presentation schedule for all groups will be provided after submitting the reports
  5. The project grade will be calculated as follows:
    • 40% Report
    • 30% Presentation
    • 30% Questions and Answers
  6. Group members shall not exceed 3 students per team
